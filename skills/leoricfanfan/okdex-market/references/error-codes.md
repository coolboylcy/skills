# OKX Market API 错误码参考

## 通用错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 0 | 成功 | - |
| 50000 | 系统错误 | 请稍后重试 |
| 50001 | 参数错误 | 检查请求参数是否正确 |
| 50002 | 签名错误 | 检查签名生成逻辑和 API 密钥 |
| 50003 | 请求频率超限 | 降低请求频率 |
| 50004 | 权限不足 | 检查 API 权限配置 |
| 50005 | 时间戳过期 | 检查时间戳是否在有效范围内 |

## 行情价格 API 错误码

## 签名错误排查

如果遇到签名错误 (50002)，请检查：

1. **时间戳格式**: 必须是 ISO 8601 格式，如 `2023-10-18T12:21:41.274Z`
2. **签名内容**: `timestamp + method + request_path + body`
   - GET 请求的 body 为空字符串
   - POST 请求的 body 为 JSON 字符串
3. **签名算法**: HMAC SHA256，结果 Base64 编码
4. **请求路径**: 包含查询参数，如 `/api/v6/dex/market/candles?chainIndex=1&...`

## 请求频率限制

- 默认限制: 10 请求/秒
- Websocket 连接限制: 3 请求/秒 (基于 API KEY)

如需更高频率，请联系 OKX 申请白名单。
